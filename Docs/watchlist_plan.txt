WATCHLIST + PRICE/SIGNAL ALERTS — IMPLEMENTATION PLAN
======================================================

CONTEXT
-------
New "Watchlist" tab in the Opes stock analysis app. Users save tickers and
receive email alerts when configurable conditions are met:
  - Price crosses a user-set threshold
  - RSI hits oversold (<30) or overbought (>70)
  - ML rating changes to Buy

Designed as a retention driver — alerts fire in the background daily.

CHOSEN STACK
------------
- Storage:    Upstash Redis (free tier: 10k commands/day, 256MB, no expiry)
- Email:      Resend (free tier: 3,000 emails/month)
- Cron:       GitHub Actions scheduled workflow (free) → POST /api/check_alerts
- Cards:      Price + RSI + ML signal (concurrent fetch, n_estimators=50)


EXTERNAL SETUP (One-Time, Done Manually Before Deploying)
----------------------------------------------------------
1. Upstash Redis — upstash.com → Create Database → copy REST URL + token
   Add to Vercel env vars: UPSTASH_REDIS_REST_URL, UPSTASH_REDIS_REST_TOKEN
   Add to local .env:      same two vars

2. Resend — resend.com → Create API key → verify sending domain
   (use onboarding@resend.dev for initial testing)
   Add to Vercel env vars: RESEND_API_KEY, RESEND_FROM_EMAIL
   Add to local .env:      same two vars

3. Cron secret — generate: openssl rand -hex 32
   Add to Vercel env vars: CRON_SECRET
   Add to GitHub repo Secrets: CRON_SECRET, APP_URL (= https://finopes.vercel.app)


REDIS KEY SCHEMA
----------------
Key                            Type          Value
------------------------------  ------------  ----------------------------------------
watchlist:{user_id}             String (JSON) [{symbol, addedAt}]
alerts:{user_id}                String (JSON) [{id, symbol, type, condition,
                                               threshold, enabled, lastTriggeredAt}]
user_email:{user_id}            String        "user@example.com"
alert_user_ids                  Redis Set     All user_ids with >= 1 active alert
alert_state:{user_id}:{symbol}  String (JSON) {lastRating, checkedAt}

Alert types:
  price:         {type:"price", condition:"above"|"below", threshold:200.0}
  rsi:           {type:"rsi", condition:"above"|"below", threshold:70}
  rating_change: {type:"rating_change"} — triggers when direction flips to bullish

api/watchlist.py
  GET  /api/watchlist  — JWT auth, fetch symbols from Redis, concurrent fetch
                         quote + 1yr prices + quick ML (n_estimators=50) per symbol
  POST /api/watchlist  — Add symbol (validate_symbol()), cap at 20 symbols
  DELETE /api/watchlist?symbol=X — Remove symbol from Redis

  Response shape (GET):
  {
    "watchlist": [
      {
        "symbol": "AAPL", "addedAt": "...", "name": "Apple Inc.",
        "price": 195.23, "change": -1.45, "change_percent": "-0.74%",
        "rsi": 52.3, "rsi_signal": "Neutral",
        "direction": "bullish", "rating": "Moderate Buy", "quality_score": 65.2
      }
    ]
  }

  Patterns to reuse:
  - ThreadPoolExecutor concurrent fetch from api/topstocks.py (cap 8 workers)
  - get_daily_prices(), get_quote(), get_company_overview(), train_and_predict()
    copied from api/analyze.py (safe duplication for MVP, avoids refactor risk)

api/alerts.py
  GET    /api/alerts         — Return all alert rules + stored email for user
  POST   /api/alerts         — Create alert, store user email, add to alert_user_ids set
  PATCH  /api/alerts?id=UUID — Toggle enabled/disabled
  DELETE /api/alerts?id=UUID — Remove; remove user from alert_user_ids if no alerts remain

api/check_alerts.py
  POST /api/check_alerts — Authenticated by X-Cron-Secret header (not JWT)
  Logic:
    1. Get all members of alert_user_ids Redis Set
    2. For each user: fetch their alerts and stored email
    3. For each enabled alert: fetch current data (quote for price, 14-day RSI,
       quick ML for rating alerts)
    4. Evaluate condition; if triggered AND lastTriggeredAt != today:
       send email via Resend, update lastTriggeredAt
    5. For rating_change alerts: compare against alert_state:{user_id}:{symbol};
       update state after check
  Returns: {checked: N, triggered: M, emails_sent: K}

.github/workflows/check_alerts.yml
  Runs at 9:30am ET (14:30 UTC) Mon–Fri + workflow_dispatch for manual testing:

  name: Check Stock Alerts
  on:
    schedule:
      - cron: '30 14 * * 1-5'
    workflow_dispatch:
  jobs:
    check-alerts:
      runs-on: ubuntu-latest
      steps:
        - name: Trigger alert check
          run: |
            curl -f -X POST ${{ secrets.APP_URL }}/api/check_alerts \
              -H "X-Cron-Secret: ${{ secrets.CRON_SECRET }}" \
              -H "Content-Type: application/json"
